About this Website!

Austin John | Oct 14, 2023 min read

How I built this website, and how you can too!

Hey, thanks for checking out this post. This will be a quick read on how I host and maintain this website! Keep in mind, this post is written with the assumption that you are familiar with programming languages, version control, and some HTML/Markdown experience. To get started, you'll need a github account, and the following software installed:

  • git
  • GO programming language
  • Hugo

Depending on what operating system you are using, you may need to install the software differently. I used a Windows computer to create and maintain this site, so I'll be explaining this in that syntax. If you need additional help, you can reach out to me, or check out the official Hugo website's documentation tab for more information.

If this is your first time using Git, you may also need to configure git by setting global email and username.

Github

To get started, you're going to want to create 2 github repositories and clone them to your computer. The first repository will store the website files where you actively edit and make changes. The other repository will store the static files of the website, which Hugo will automatically generate when you build it with the Hugo command.

  1. Static File Repository
    • When making this repository, name it accordingly with the following ending. For me as it was a personal website, I named the repository "austinj1522.github.io". Clone this repository to your local computer.
  2. Active Repository
    • When making this repository, name it whatever you'd like! For me, I named it "website" to make organization easier on my end. Clone this repository to your local computer.

Hugo

Hugo makes building websites incredibly easy. To start, first open a command prompt window and navigate to your active repository, in my case “website”. Run the following command

hugo new site WebsiteNameHere

Next, reopen the window which contains the theme you selected. Copy the github clone link. Change directory to the WebsiteNameHere directory, and then run the following commands.

cd themes

git clone GithubLinkHere

Your new website theme can now use the pre-built theme you selected. Just a couple more steps…

Open your website and find the config.yaml file. If it doesn’t have one, create one! This file will be used to configure the website, and in this case will specify the theme for the site. To do this, open the file and create a new line below “title” Specify the theme by entering the following (my theme’s name is hugo-profile):

theme: hugo-profile

Some themes may have slightly different syntaxes, be sure to check out the ReadMe of the theme to find out more about variations!

Your header of your config.yaml file should look similar to the following, except with your repo and theme names:

baseURL: “https://austinj1522.github.io”
languageCode: “en-us”
title: “Austin’s Website”
theme: hugo-profile

Your base website should now be running with the theme you selected. To test this, navigate to the WebsiteNameHere repository in your command prompt and run the following command:

hugo server

You should see it build and then deploy the site on localhost:1313, may be different in some cases.

Connect Repositories

Now that the website has been created, you can begin to add your information and customize it however you’d like. The last step is to connect the repositories so that whenever you build the website, it automatically deploys on Github Pages. To do this, we will need to add a git submodule.

  1. Open command prompt and naviagate to WebsiteNameHere repository.

    • Run the following command:

      git submodule add -b main githubLinkHere public

      • “main” refers to the branch of the repository you will be committing
      • githubLinkHere should be the github clone link of the Static File Repository you created earlier. Keep in mind, we haven’t touched or used this repository yet apart from cloning it to our local computer and initializing it.
      • “public” specifies the folder that will be the folder you are deploying from. “public” is the name of the folder that gets automatically generated whenever you build the website, so that’s why we specify “public”.

To confirm if you have correctly connected the repositories, check the git origins of the public folder, and of your WebsiteNameHere folder. The public folder should be pulling/pushing to your static file repository, and your WebsiteNameHere folder should be Active Repository

Please make sure you check the ReadME and intructions of the prebuilt theme you select. Not all of them work the same way and some have slight variations in syntax, so to avoid errors, read ahead!
